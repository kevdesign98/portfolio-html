L'obbiettivo

Come sviluppatore junior, volevo mettermi alla prova con un framework solido come Angular. La sfida è stata capire come far comunicare correttamente l'applicazione con delle API esterne e gestire la visualizzazione dei dati in modo ordinato ed efficiente."

Soluzione

Ho scelto Angular 18 per iniziare subito con le ultime convenzioni del framework (come i componenti standalone).
Per la parte visiva, ho preferito Tailwind CSS per la velocità di styling e PrimeNG per avere componenti già pronti e professionali, concentrandomi così maggiormente sulla logica di recupero dei dati (HTTP Client).

Sviluppo & Performance

Nello sviluppo di Val-App ho dato priorità all'organizzazione del codice tramite i componenti standalone di Angular 18, rendendo l'app modulare e facile da scalare. Ho utilizzato Tailwind CSS per mantenere il bundle finale ridotto e PrimeNG per avere componenti UI già ottimizzati. Mi sono divertito a gestire le chiamate alle API esterne cercando di rendere l'esperienza utente il più fluida possibile, gestendo i caricamenti in modo che l'interfaccia non risulti mai 'bloccata' per chi la usa
